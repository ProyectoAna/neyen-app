About the NEYĒN System Dashboard

What is this?

This is the NEYĒN System Metadata Dashboard — a live, external, queryable view of the entire NEYĒN transformation system infrastructure.

Unlike traditional consultancy deliverables, this isn't a static PDF or PowerPoint deck. It's a persistent system of record that exists independently of any conversation or documentation.

What does it show?

The dashboard displays 17 interconnected entries spanning from 1974 to present, organized by:

  • Symbols: Core concepts like water consciousness and serpent-star geometry
  • Systems: Operational architectures like coherence protocols and cross-linking engines
  • Pilots: Verified transformation projects (HELIOS, Xmas, Paterna)
  • Origins: Foundational history (DINIBA 1974, "no romper" philosophy)
  • Legal framework: Trademark, publication, company registration

Why does it exist?

Most organizational transformation systems live in people's heads or scattered documents. NEYĒN infrastructure is:

  • External: Stored in Supabase, accessible via API endpoint
  • Queryable: Any AI or system can read the complete metadata
  • Connected: 16/17 entries cross-linked by shared tags, entities, and narrative roles
  • Evolving: Auto-updates as new content ingests via Slack, Telegram, or manual workflows
For clients: Your transformation doesn't disappear when the consultant leaves. It lives as infrastructure that grows with you.

How to use it

Filter by narrative role: Click role buttons (Symbol, System, Pilot, etc.) to see specific entry types.

View connections: Each entry shows related entries via tags and entities.

Explore timeline: Entries span 1974-present, showing evolution from DINIBA's founding through current pilots.

Check sources: All data comes from verified sources (gamma_presentation, website, manual, verified_memory).

Technical details

Endpoint: https://n8n-docker-t9sr.onrender.com/webhook/system

Infrastructure: n8n workflows → Supabase PostgreSQL → JSON API → Dashboard

Cross-linking: Automated scoring algorithm (tags × 3 pts, entities × 2 pts, role × 1 pt)

Updates: Dashboard auto-refreshes every 5 minutes
